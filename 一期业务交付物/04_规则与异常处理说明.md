# 规则与异常处理说明

## 1. 员工分流规则

一期默认员工分流策略：

- 渠道活码绑定一个员工池。
- 员工池内仅启用状态的员工参与分流。
- 客户扫码添加员工后，以实际添加成功的员工为最终承接员工。
- 停用员工、异常员工不再继续承接新客户。
- 员工不可用时记录日志并触发告警。

建议业务口径：

| 场景 | 处理方式 |
| --- | --- |
| 员工池有多个启用员工 | 按均衡承接原则分配客户 |
| 员工被停用 | 不再分配新客户，历史客户关系保留 |
| 员工异常 | 跳过该员工并触发告警 |
| 员工池无可用员工 | 该渠道活码进入异常状态并触发告警 |

---

## 2. 分群规则优先级

一期分群优先级固定为：

```text
客户标签规则
>
渠道标签规则
>
渠道默认群池
>
全局默认群池
```

含义：

| 优先级 | 规则 | 说明 |
| --- | --- | --- |
| 1 | 客户标签规则 | 例如 VIP客户进入 VIP 群池 |
| 2 | 渠道标签规则 | 例如抖音线索进入抖音客户群池 |
| 3 | 渠道默认群池 | 渠道配置中绑定的默认群池 |
| 4 | 全局默认群池 | 所有规则都未命中时兜底 |

规则要求：

- 规则可启用 / 停用。
- 规则可设置优先级。
- 匹配结果需要记录日志。
- 同时命中多个规则时，以最高优先级规则为准。

---

## 3. 群满切换规则

| 场景 | 处理方式 |
| --- | --- |
| 当前群未达到阈值 | 继续发送当前群入口 |
| 当前群达到阈值 | 切换到下一个可用备用群 |
| 多个备用群可用 | 按群池中配置的排序选择 |
| 备用群也达到阈值 | 继续寻找下一个可用备用群 |
| 所有备用群不可用 | 触发告警 |
| 配置了自动建群 | 标记为后续扩展流程，一期先保留配置口径 |

业务要求：

- 群人数达到阈值后，不再优先承接后续新客户。
- 群满切换需要记录日志。
- 群池无可用群时必须告警。

---

## 4. 默认兜底策略

| 异常或缺失场景 | 兜底处理 |
| --- | --- |
| 无法识别渠道 | 标记为未知渠道，并使用全局默认配置 |
| 渠道未绑定欢迎语 | 使用全局默认欢迎语 |
| 渠道未绑定群池 | 使用全局默认群池 |
| 未匹配任何分群规则 | 使用全局默认群池 |
| 无可用员工 | 告警，不继续分配新客户 |
| 无可用客户群 | 告警，欢迎语仍可发送但不附带有效入群入口 |
| 标签打标失败 | 记录失败日志并告警 |
| 欢迎语发送失败 | 记录失败日志并告警 |
| 客户进群事件无法匹配客户 | 记录异常日志，进入待人工排查状态 |
| 客户退群事件无法匹配客户 | 记录异常日志，进入待人工排查状态 |

---

## 5. 异常处理分级

| 等级 | 场景 | 处理要求 |
| --- | --- | --- |
| 高 | 无可用员工、无可用群、回调异常、关键消息发送失败 | 立即告警，运营或管理员处理 |
| 中 | 标签同步失败、群人数达到阈值、数据同步失败 | 告警并进入待处理列表 |
| 低 | 单条日志记录异常、非关键素材不可用 | 记录日志，后台可查询 |

---

## 6. 人工处理要求

- 告警中心需要支持标记已处理。
- 高等级告警需要记录处理人和处理备注。
- 运营可通过日志中心查询客户完整链路。
- 对于无法自动修复的异常，应保留人工调整入口，例如手动调整客户标签、停用员工、停用群、切换默认群池。
