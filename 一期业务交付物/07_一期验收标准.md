# 一期验收标准

## 1. 验收准备

验收前应准备以下业务数据：

- 至少 2 个渠道，例如抖音、小红书。
- 至少 1 个员工池，包含 2 名以上启用员工。
- 至少 2 个欢迎语模板。
- 至少 1 组文本、图片、链接或文件素材。
- 至少 2 个渠道标签，例如抖音线索、小红书线索。
- 至少 1 个群池，包含 2 个以上客户群。
- 至少 1 条客户标签分群规则。
- 至少 1 条渠道分群规则。
- 至少 1 个全局默认群池。
- 至少 1 个全局默认欢迎语。

---

## 2. 渠道活码验收

| 验收项 | 通过标准 |
| --- | --- |
| 创建渠道活码 | 后台可创建新活码 |
| 编辑渠道活码 | 可修改渠道名称、标签、员工池、欢迎语、群池 |
| 启停渠道活码 | 可启用和停用 |
| 二维码展示 | 后台可查看二维码 |
| 二维码复制 / 下载 | 可复制或下载用于投放 |
| 渠道绑定 | 活码可绑定渠道标签、员工池、欢迎语、默认群池 |

---

## 3. 客户加好友验收

| 验收项 | 通过标准 |
| --- | --- |
| 客户扫码 | 客户可通过投放二维码进入添加员工流程 |
| 添加员工 | 客户可成功添加企业微信员工 |
| 来源识别 | 系统可识别客户来自哪个渠道 |
| 员工记录 | 系统可记录客户实际添加的员工 |
| 客户档案 | 系统可创建或更新客户记录 |
| 事件日志 | 客户添加链路有完整日志 |

---

## 4. 欢迎语验收

| 验收项 | 通过标准 |
| --- | --- |
| 渠道欢迎语 | 不同渠道可收到对应欢迎语 |
| 默认欢迎语 | 未匹配渠道时使用默认欢迎语 |
| 文本内容 | 欢迎语可包含文本 |
| 客户昵称变量 | 欢迎语可使用客户昵称变量 |
| 素材引用 | 欢迎语可引用图片、链接、文件素材 |
| 入群入口 | 欢迎语可附带入群二维码或入群入口 |
| 发送日志 | 成功和失败均有日志 |

---

## 5. 自动打标验收

| 验收项 | 通过标准 |
| --- | --- |
| 渠道标签 | 客户添加好友后自动获得渠道标签 |
| 群标签 | 客户进群后自动获得群标签 |
| 手动调整 | 后台可手动调整客户标签 |
| 标签日志 | 标签变化有日志 |
| 失败告警 | 打标失败时产生日志和告警 |

---

## 6. 群池分流验收

| 验收项 | 通过标准 |
| --- | --- |
| 创建群池 | 后台可创建群池 |
| 绑定客户群 | 一个群池可绑定多个客户群 |
| 渠道默认群池 | 渠道可绑定默认群池 |
| 规则匹配 | 客户可按标签或渠道匹配目标群池 |
| 优先级 | 客户标签规则优先于渠道规则和默认群池 |
| 群满切换 | 当前群达到阈值后切换备用群 |
| 无可用群告警 | 所有备用群不可用时触发告警 |

## 7. 进群 / 退群验收

| 验收项 | 通过标准 |
| --- | --- |
| 客户进群记录 | 客户进群后记录客户、群、时间、渠道 |
| 客户退群记录 | 客户退群后记录退群时间 |
| 群关系状态 | 可查看客户是否仍在群内 |
| 群人数更新 | 客户进退群后更新群人数 |
| 转化数据更新 | 客户进群后更新渠道转化数据 |
| 事件日志 | 进群和退群均有日志 |

## 8. 统计报表验收

| 验收项 | 通过标准 |
| --- | --- |
| 新增好友数 | 可按时间和渠道查看 |
| 成功进群人数 | 可按时间和渠道查看 |
| 退群人数 | 可按时间和渠道查看 |
| 进群转化率 | 按成功进群人数 / 新增好友数计算 |
| 员工承接量 | 可查看每个员工承接客户数 |
| 群池容量 | 可查看群人数和剩余容量 |
| 欢迎语成功率 | 可查看成功数、失败数和成功率 |
| 异常数 | 可查看接口和业务异常数量 |
| 导出 | 可按筛选条件导出 CSV / Excel |

## 9. 日志与告警验收

| 验收项 | 通过标准 |
| --- | --- |
| 日志中心 | 可查看关键业务链路日志 |
| 日志筛选 | 可按时间、类型、客户、员工、渠道、群筛选 |
| 告警中心 | 可查看异常告警 |
| 告警通知 | 高等级告警可通过后台、企业微信内部群机器人、邮件通知 |
| 告警处理 | 可标记处理中、已处理、已忽略 |
| 处理记录 | 处理人、处理时间、处理备注可追踪 |

---

## 10. 端到端验收

完整链路必须通过：

```text
运营创建渠道活码
↓
客户扫码并添加员工
↓
系统识别渠道和员工
↓
系统创建客户档案
↓
系统自动打渠道标签
↓
系统匹配欢迎语和群池
↓
客户收到欢迎语和入群入口
↓
客户自主进入客户群
↓
系统记录进群关系
↓
系统更新标签、群人数和统计
↓
客户退群后系统记录退群关系
↓
后台可查看统计、日志和告警
```

通过标准：

- 主链路可完整走通。
- 每个关键节点都有记录。
- 异常场景能产生日志或告警。
- 后台可完成运营日常配置和查看。
- 未实现任何一期明确不做的高风险能力。
