# 日志与告警清单

## 1. 日志类型

| 日志类型 | 记录时机 | 主要用途 |
| --- | --- | --- |
| 活码创建日志 | 运营创建或更新渠道活码 | 追踪配置变更 |
| 客户扫码 / 添加好友日志 | 客户添加员工成功后 | 追踪客户来源和承接员工 |
| 企业微信回调日志 | 收到企业微信事件时 | 排查事件处理问题 |
| 客户打标签日志 | 自动或手动调整标签时 | 追踪标签变化 |
| 欢迎语发送日志 | 发送欢迎语后 | 追踪发送成功或失败 |
| 入群入口发送日志 | 发送入群二维码或链接后 | 追踪客户入群引导 |
| 客户进群日志 | 客户进入客户群后 | 追踪转化结果 |
| 客户退群日志 | 客户退出客户群后 | 追踪流失情况 |
| 分群规则匹配日志 | 系统匹配群池时 | 解释客户为什么进入某群池 |
| 群池切换日志 | 当前群达到阈值并切换时 | 追踪容量变化 |
| 外部能力调用日志 | 调用企业微信相关能力后 | 追踪成功或失败 |
| 异常告警日志 | 触发告警时 | 追踪异常处理 |

---

## 2. 日志字段

| 字段 | 说明 |
| --- | --- |
| 日志 ID | 日志唯一标识 |
| 日志类型 | 活码、客户、回调、标签、欢迎语、入群、退群、告警等 |
| 业务对象 ID | 关联的活码、客户、群池、规则等对象 |
| 客户 external_userid | 客户标识 |
| 员工 userid | 承接员工标识 |
| 群 chat_id | 客户群标识 |
| 渠道 ID | 客户来源渠道 |
| 请求参数 | 关键请求内容 |
| 响应结果 | 关键响应内容 |
| 错误信息 | 失败原因 |
| 创建时间 | 日志产生时间 |

---

## 3. 告警场景

| 告警场景 | 等级 | 处理对象 |
| --- | --- | --- |
| 企业微信相关能力调用失败 | 高 | 管理员 |
| 欢迎语发送失败 | 高 | 运营 / 管理员 |
| 客户打标签失败 | 中 | 运营 |
| 群二维码失效 | 高 | 运营 / 管理员 |
| 群人数达到阈值 | 中 | 运营 |
| 群池无可用备用群 | 高 | 运营 / 管理员 |
| 员工不可用 | 高 | 运营 / 管理员 |
| access_token 获取失败 | 高 | 管理员 |
| 回调解密失败 | 高 | 管理员 |
| 数据同步失败 | 中 | 管理员 |

---

## 4. 告警方式

| 方式 | 一期状态 | 说明 |
| --- | --- | --- |
| 后台告警列表 | 必须支持 | 所有告警统一沉淀 |
| 企业微信内部群机器人通知 | 必须支持 | 高等级告警及时通知 |
| 邮件通知 | 必须支持 | 管理员和运营可接收 |
| 短信通知 | 后续扩展 | 不进入一期默认范围 |

---

## 5. 告警处理状态

告警中心至少支持以下状态：

```text
未处理
处理中
已处理
已忽略
```

处理要求：

- 告警应显示触发时间、告警类型、告警等级、关联对象、错误信息。
- 告警处理后应记录处理人、处理时间、处理备注。
- 高等级告警不应只记录日志，必须进入通知链路。

---

## 6. 日志查询要求

- 支持按时间范围查询。
- 支持按客户、员工、渠道、客户群查询。
- 支持按日志类型查询。
- 支持按成功 / 失败状态查询。
- 关键失败日志应能跳转或关联到对应告警。
