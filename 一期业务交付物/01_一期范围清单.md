# 一期范围清单

## 1. 一期目标范围

一期围绕“渠道引流、加好友、欢迎语、引导入群、打标、分群、统计、告警”形成完整运营闭环。

| 模块 | 一期目标 | 完成结果 |
| --- | --- | --- |
| 渠道活码管理 | 区分客户来源并生成投放二维码 | 运营可创建、编辑、启停、删除、查看、复制、下载渠道活码 |
| 员工池管理 | 多员工承接客户，降低单员工承接压力 | 运营可维护员工和员工池，渠道活码可绑定员工池 |
| 客户扫码加好友 | 识别客户来源和实际承接员工 | 客户添加员工后形成客户档案、客户员工关系和事件记录 |
| 新客户欢迎语 | 客户加好友后收到标准化欢迎内容 | 不同渠道可配置不同欢迎语，未匹配时使用默认欢迎语 |
| 素材库管理 | 统一维护欢迎语可引用的素材 | 文本、图片、文件、链接素材可维护、启停和引用 |
| 客户标签管理 | 根据渠道和进群行为形成客户标识 | 支持渠道标签、客户分层标签、群标签和手动调整 |
| 群池管理 | 多个客户群承接同类客户 | 渠道可绑定群池，群满后切换备用群 |
| 分群规则 | 按渠道和标签决定客户进入哪个群池 | 支持规则优先级、启停、默认兜底 |
| 进群 / 退群记录 | 记录客户与客户群的关系变化 | 可查看客户入群、退群、仍在群内等状态 |
| 统计报表 | 给运营提供基础转化数据 | 可按时间、渠道、员工、群池筛选并导出 |
| 日志中心 | 保留关键业务链路证据 | 可追踪活码、回调、打标、欢迎语、分群、进退群等事件 |
| 告警机制 | 异常时及时通知运营或管理员 | 后台告警列表、企业微信内部群机器人、邮件通知 |

---

## 2. 一期不做范围

| 不做事项 | 原因 |
| --- | --- |
| 微信 App 开发 | 本系统不替代微信客户端 |
| 企业微信客户端开发 | 员工继续使用官方企业微信 |
| 非官方协议 | 风险高，不符合项目边界 |
| PC Hook | 风险高，不符合项目边界 |
| 模拟点击 | 风险高，不符合项目边界 |
| 强制客户进群 | 企业微信能力和用户体验均不支持强制入群 |
| 群内 AI 智能体自动回复 | 不属于一期目标 |
| 智能体自动加入客户群 | 边界不清晰，暂不进入一期 |
| 模拟员工自动私聊 | 风险高，暂不进入一期 |
| 未经客户触发的大规模自动消息推送 | 易触发合规和平台限制 |
| 精准统计只扫码未加好友行为 | 需要额外中转落地页或短链系统 |
| 短信告警 | 作为后续扩展 |

---

## 3. 关键边界

- 系统只能引导客户入群，不能强制客户入群。
- 客户必须主动扫码、添加员工或点击入群入口。
- 员工继续通过企业微信跟进客户。
- 运营人员通过 Web 后台配置渠道、员工池、群池、话术、标签、分流规则、统计和告警。
- 一期统计重点是“加好友后”的链路数据，不默认统计“只扫码未加好友”的行为。

---

## 4. 范围冻结标准

满足以下条件后，一期业务范围视为冻结：

- 必做模块和不做模块已经确认。
- 每个后台页面的业务动作已经确认。
- 客户扫码、加好友、欢迎语、入群、进退群流程已经确认。
- 分群、群满切换、默认兜底和异常处理规则已经确认。
- 统计指标、日志类型和告警场景已经确认。
