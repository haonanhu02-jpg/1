# 企业微信自动加好友与自动入群 SCRM 系统一期需求文档

## 1. 项目背景

本项目目标是开发一个基于企业微信官方 API 的 SCRM 自动化运营系统。

系统不开发微信 App，也不开发企业微信客户端，不使用非官方协议、不做 PC Hook、不做模拟点击。客户仍然使用微信扫码、添加企业微信员工、接收欢迎语、进入客户群；员工仍然使用企业微信跟进客户；运营人员通过我们自己开发的 Web 后台配置渠道活码、群池、话术、标签、分流规则、统计数据和异常告警。

一期主要实现：

* 渠道活码管理
* 客户扫码添加企业微信员工
* 新客户欢迎语自动发送
* 入群二维码 / 入群入口自动发送
* 渠道标签自动打标
* 群池分流
* 客户进群 / 退群记录
* 后台配置管理
* 数据统计
* 异常日志与告警

本期暂不实现客户群内 AI 智能体自动回复、智能体自动加入客户群、模拟员工自动私聊等高风险或边界不清晰的功能。

---

## 2. 系统定位

本项目本质是：

```text
Web 管理后台
+ 后端服务
+ 企业微信官方 API
+ Webhook 回调服务
+ 数据库
+ 规则引擎
+ 统计报表
```

企业微信 / 微信在本项目中的角色是客户触达端和员工沟通端。

我们自己开发的系统包括：

```text
1. Web 管理后台
   用于运营人员配置渠道、活码、员工池、群池、话术、标签、规则、统计等。

2. 后端服务
   用于调用企业微信 API、接收企业微信回调、执行业务规则、保存数据、触发告警。

3. 数据库
   用于保存客户、渠道、活码、员工池、客户群、标签、分流规则、日志和统计数据。
```

## 3. 一期功能范围

### 3.1 渠道活码管理

后台支持创建渠道活码，每个活码用于区分不同客户来源，例如：

* 抖音
* 小红书
* 官网
* 线下展会
* 海报投放
* 短视频引流

功能要求：

1. 新建渠道活码。
2. 编辑渠道活码。
3. 启用 / 停用渠道活码。
4. 删除渠道活码。
5. 每个活码绑定渠道名称。
6. 每个活码绑定渠道标签。
7. 每个活码绑定一个或多个企业微信员工。
8. 每个活码绑定默认欢迎语模板。
9. 每个活码绑定默认群池。
10. 系统调用企业微信“联系我”相关接口生成二维码。
11. 保存企业微信返回的二维码地址、配置 ID、渠道参数。
12. 二维码可在后台查看、复制、下载。

核心数据：

```text
渠道名称
渠道编码
渠道标签
绑定员工
欢迎语模板
默认群池
企业微信 config_id
二维码 URL
启用状态
创建时间
更新时间
```

---

### 3.2 员工池管理

为了避免单个员工加好友达到上限，需要支持多个员工分流。

功能要求：

1. 后台维护员工列表。
2. 后台创建员工池。
3. 一个员工池可以包含多个企业微信员工。
4. 渠道活码可以绑定员工池。
5. 后端根据配置把客户分配给对应员工。
6. 记录客户实际添加的是哪个企业微信员工。
7. 支持员工启用 / 停用。
8. 员工异常时记录日志并告警。

核心数据：

```text
员工 userid
员工名称
手机号 / 备注
所属部门
是否启用
员工池 ID
员工池名称
员工池成员关系
```

---

### 3.3 客户扫码加好友处理

客户通过微信扫描渠道活码后，添加指定企业微信员工为外部联系人。

业务要求：

1. 客户扫码渠道活码。
2. 客户添加企业微信员工。
3. 企业微信向系统回调新增外部联系人事件。
4. 系统解析回调中的 external_userid、userid、state、welcome_code 等信息。
5. 系统根据 state 判断客户来源渠道。
6. 系统保存客户基本信息。
7. 系统保存客户与员工的关系。
8. 系统给客户自动打渠道标签。
9. 系统匹配欢迎语模板。
10. 系统匹配对应群池。
11. 系统调用企业微信新客户欢迎语接口，发送欢迎语和入群入口。
12. 系统记录完整事件日志。

注意：

* 系统不能强制客户进群。
* 系统只能自动发送入群二维码、入群链接或入群引导。
* 客户需要自行点击或扫码进入客户群。

---

### 3.4 新客户欢迎语管理

客户添加企业微信员工成功后，系统自动发送欢迎语。

功能要求：

1. 后台可创建欢迎语模板。
2. 欢迎语模板支持文本。
3. 欢迎语模板支持客户昵称变量。
4. 欢迎语模板支持图片素材。
5. 欢迎语模板支持链接素材。
6. 欢迎语模板支持文件 / 资料包。
7. 欢迎语模板支持入群二维码。
8. 不同渠道可绑定不同欢迎语。
9. 未匹配到渠道时使用默认欢迎语。
10. 欢迎语发送成功 / 失败需要记录日志。

示例欢迎语：

```text
你好，欢迎咨询我们的服务。
这是你的专属资料群入口，请点击下方二维码加入客户群。
如有问题，也可以直接在这里留言。
```

---

### 3.5 素材库管理

后台统一维护欢迎语、图片、文件、链接等素材。

功能要求：

1. 创建文本素材。
2. 上传图片素材。
3. 上传文件素材。
4. 创建链接素材。
5. 素材可启用 / 停用。
6. 素材可被欢迎语模板引用。
7. 素材可被群规模板引用。
8. 记录素材创建人、创建时间、更新时间。

核心数据：

```text
素材名称
素材类型
素材内容
素材 URL
文件大小
启用状态
创建人
创建时间
```

---

### 3.6 客户标签管理

系统根据渠道、客户行为、进群情况给客户自动打标签。

功能要求：

1. 后台维护标签。
2. 支持渠道标签。
3. 支持客户分层标签。
4. 支持群标签。
5. 客户添加好友后，根据渠道自动打标签。
6. 客户进入指定群后，自动打群标签。
7. 支持后台手动调整客户标签。
8. 标签变化需要记录日志。
9. 标签需要同步到本地数据库。
10. 如需同步到企业微信，则调用企业微信客户标签接口。

示例标签：

```text
抖音线索
小红书线索
官网线索
线下展会
已进群
未进群
高意向客户
VIP客户
```

---

### 3.7 群池管理

一个渠道可以绑定一个群池，一个群池可以包含多个企业微信客户群。

功能要求：

1. 后台创建群池。
2. 群池绑定多个企业微信客户群。
3. 支持设置群池名称。
4. 支持设置群池对应渠道。
5. 支持设置群满切换阈值。
6. 支持设置默认备用群。
7. 支持查看每个群当前人数。
8. 支持同步企业微信群详情。
9. 当前群达到阈值后，后续客户切换到下一个备用群。
10. 如果所有备用群不可用，触发告警。
11. 可配置是否自动创建新群。
12. 客户进入群后，记录客户与群的关系。

核心数据：

```text
群池 ID
群池名称
渠道 ID
客户群 chat_id
客户群名称
群主 userid
当前人数
人数阈值
群状态
排序
是否备用群
```

---

### 3.8 分群规则

系统根据渠道和标签决定客户应该进入哪个群池。

规则优先级建议：

```text
客户标签规则 > 渠道标签规则 > 渠道默认群池 > 全局默认群池
```

功能要求：

1. 后台创建分群规则。
2. 支持按渠道分群。
3. 支持按客户标签分群。
4. 支持渠道 + 客户标签组合分群。
5. 支持默认兜底群池。
6. 规则可启用 / 停用。
7. 规则可设置优先级。
8. 匹配结果需要记录日志。

示例规则：

```text
规则 1：
如果客户标签 = VIP客户
则分配到 VIP 群池

规则 2：
如果渠道 = 抖音
则分配到 抖音客户群池

规则 3：
如果渠道 = 小红书
则分配到 小红书客户群池

规则 4：
如果没有匹配任何规则
则分配到 默认客户群池
```

---

### 3.9 客户进群 / 退群记录

客户进入或退出客户群后，企业微信回调系统，系统记录客户群关系。

功能要求：

1. 接收企业微信客户群变更事件。
2. 识别客户进群事件。
3. 识别客户退群事件。
4. 同步客户群详情。
5. 保存客户与群的关系。
6. 保存进群时间。
7. 保存退群时间。
8. 保存入群渠道。
9. 客户进群后自动打群标签。
10. 更新渠道转化数据。
11. 更新群池人数。
12. 记录事件日志。

核心数据：

```text
客户 external_userid
客户名称
群 chat_id
群名称
入群时间
退群时间
入群渠道
入群方式
是否仍在群内
```

---

### 3.10 统计报表

后台提供基础统计数据。

功能要求：

1. 统计每个渠道新增客户数。
2. 统计每个渠道成功进群人数。
3. 统计每个渠道退群人数。
4. 统计每个渠道进群转化率。
5. 统计每个员工承接客户数。
6. 统计每个群当前人数。
7. 统计欢迎语发送成功率。
8. 统计 API 调用失败次数。
9. 支持按时间范围筛选。
10. 支持导出 CSV / Excel。

基础指标：

```text
新增好友数
成功进群人数
退群人数
进群转化率
员工承接量
群池剩余容量
欢迎语发送成功数
欢迎语发送失败数
接口异常数
```

说明：

如果需要精准统计“扫码量”，建议额外开发中转落地页或短链系统：

```text
客户扫描我们自己的统计二维码
↓
系统记录一次访问
↓
页面展示或跳转企业微信联系我二维码
```

如果直接使用企业微信联系我二维码，系统可能无法完整记录“只扫码但未添加好友”的客户行为。

### 3.11 日志中心

系统需要记录完整链路日志。

日志类型：

1. 活码创建日志。
2. 客户扫码 / 添加好友日志。
3. 企业微信回调日志。
4. 客户打标签日志。
5. 欢迎语发送日志。
6. 入群入口发送日志。
7. 客户进群日志。
8. 客户退群日志。
9. 分群规则匹配日志。
10. 群池切换日志。
11. API 调用成功 / 失败日志。
12. 异常告警日志。

日志字段：

```text
日志 ID
日志类型
业务对象 ID
客户 external_userid
员工 userid
群 chat_id
渠道 ID
请求参数
响应结果
错误信息
创建时间
```

---

### 3.12 告警机制

系统出现异常时，需要通知管理员或运营人员。

告警场景：

1. 企业微信 API 调用失败。
2. 欢迎语发送失败。
3. 客户打标签失败。
4. 群二维码失效。
5. 群人数达到阈值。
6. 群池无可用备用群。
7. 员工不可用。
8. 企业微信 access_token 获取失败。
9. 回调解密失败。
10. 数据同步失败。

告警方式：

1. 后台告警列表。
2. 企业微信内部群机器人通知。
3. 邮件通知。
4. 短信通知，可作为后续扩展。

## 4. 业务流程

### 4.1 运营创建渠道活码流程

```text
运营登录后台
↓
新建渠道活码
↓
填写渠道名称，例如“抖音”
↓
绑定员工池，例如“销售一组”
↓
绑定渠道标签，例如“抖音线索”
↓
绑定欢迎语模板
↓
绑定群池，例如“抖音客户群池”
↓
后端调用企业微信 API 创建联系我二维码
↓
企业微信返回二维码地址和配置 ID
↓
系统保存配置
↓
后台展示二维码
↓
运营把二维码投放到对应渠道
```

---

### 4.2 客户扫码加好友流程

```text
客户微信扫码
↓
进入添加企业微信员工页面
↓
客户添加员工
↓
企业微信触发新增外部联系人事件
↓
企业微信回调我们的后端
↓
后端解析 external_userid、userid、state、welcome_code
↓
系统根据 state 判断渠道来源
↓
系统创建或更新客户记录
↓
系统记录客户与员工关系
↓
系统给客户打渠道标签
↓
系统匹配欢迎语模板
↓
系统匹配对应群池
↓
系统获取当前可用客户群二维码
↓
系统调用欢迎语接口发送欢迎语和入群入口
↓
客户微信收到欢迎语和入群二维码
↓
系统记录完整日志
```

---

### 4.3 客户进群流程

```text
客户点击或扫描欢迎语中的入群二维码
↓
客户进入对应客户群
↓
企业微信触发客户群变更事件
↓
企业微信回调我们的后端
↓
后端解析 chat_id、external_userid、变更类型
↓
系统同步客户群详情
↓
系统记录客户已进群
↓
系统保存客户与群关系
↓
系统给客户打“已进群”标签
↓
系统更新渠道转化统计
↓
系统更新群池人数
↓
系统记录进群日志
```

---

### 4.4 客户退群流程

```text
客户退出客户群
↓
企业微信触发客户群变更事件
↓
企业微信回调我们的后端
↓
后端解析退群客户 external_userid 和 chat_id
↓
系统更新客户与群关系
↓
系统记录退群时间
↓
系统更新群池人数
↓
系统更新退群统计
↓
系统记录退群日志
```

---

### 4.5 群满切换流程

```text
系统同步客户群人数
↓
判断当前群人数是否达到配置阈值
↓
如果未达到阈值，继续使用当前群
↓
如果达到阈值，切换到下一个备用群
↓
如果存在备用群，后续客户发送备用群二维码
↓
如果没有备用群，触发告警
↓
如果配置了自动建群，则调用企业微信接口创建新群或新的进群方式
```

---

### 4.6 后台统计查看流程

```text
↓

↓
选择渠道 / 员工 / 群池
↓
系统查询客户、群关系、日志数据
↓
展示新增好友数、进群人数、退群人数、进群转化率
↓
支持导出数据
```
